## Sun 响应式样式方案（规划稿）

目标：在手机/平板/桌面上保持一致的品牌体验与可读性，同时优化信息层级与交互反馈，避免在不同尺寸上出现拥挤或空洞。

### 设计原则
- 以内容为中心：优先保证标题、主 CTA、关键数据/功能模块在首屏可见。
- 以流体布局为主：容器宽度、间距与字体在不同断点平滑变化，减少跳变。
- 以可访问性为底线：对比度、字号、触控尺寸满足基本可用标准。
- 以可维护为导向：使用统一的间距/字号/色彩尺度，减少“散点式”自定义。

### 断点与布局策略
- 断点建议（可按 Tailwind 默认）：`sm` 640、`md` 768、`lg` 1024、`xl` 1280、`2xl` 1536。
- 布局规则：
  - `sm` 以下：单列、内容优先，卡片与按钮全宽。
  - `md`：关键区域允许 2 列，次要内容下移或折叠。
  - `lg`：核心区域 2-3 列，侧栏/工具区出现。
  - `xl+`：限制最大内容宽度，避免过宽造成阅读疲劳。

### 栅格与容器
- 容器宽度：`max-w-screen-xl` 或 `2xl`，中心对齐，左右留白随屏幕扩大。
- 栅格：`grid` + `gap` 作为基础，避免绝对定位。
- 间距尺度：统一使用 4/8/12/16/24/32 递进，形成节奏。

### 字体与排版
- 字号策略：
  - 标题：`text-2xl` → `text-4xl` 随断点放大。
  - 正文：`text-sm`/`text-base`，行高 `leading-relaxed`。
  - 辅助信息：`text-xs`/`text-sm`，颜色减弱。
- 行宽：正文段落 `max-w-prose` 或控制字长 60-80 字符。

### 颜色与主题
- 主色/强调色/中性色三层级：
  - 主色用于 CTA 与关键高亮。
  - 强调色用于提示/状态（成功/警告/错误）。
  - 中性色用于背景与分割。
- 背景对比：
  - 主背景与卡片背景有轻微层级差（亮度/阴影）。

### 组件与交互
- 卡片：统一圆角、阴影和内边距，响应式调整列数。
- 按钮：
  - 触控区域 ≥ 44px。
  - 主要/次要按钮色彩分层，悬停/按下状态清晰。
- 导航：
  - 移动端使用汉堡菜单或底部导航。
  - 桌面端展示完整菜单与二级入口。
- 图表/媒体：
  - 尽量使用流体宽度，固定高宽比。
  - 超小屏可切换为摘要模式或滑动查看。

### 图片与多媒体
- 使用 `next/image` 的响应式尺寸配置。
- 关键图像优先加载，非首屏可延迟加载。

### 可访问性
- 对比度至少 4.5:1。
- 焦点态清晰可见。
- 表单错误提示清晰可理解。

### Tailwind 实现建议
- 在 `globals.css` 建立基础排版与颜色 token。
- 使用 `@layer base` 配置 `body`、`h1-h3`、`p` 的基础样式。
- 使用 `@layer components` 提供可复用的卡片、按钮、布局类。

### 后续落地步骤（建议）
1. 盘点页面结构与核心模块，确认信息层级。
2. 为关键组件定义统一样式（按钮、卡片、导航）。
3. 根据断点逐步调整布局与字体。
4. 进行多设备验证与可访问性检查。
